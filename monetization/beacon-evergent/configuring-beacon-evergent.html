Introduction

Configuring Beacon and Evergent to work together requires some setup in both systems. In several instances, you need to copy information from one system to the other, or replicate information exactly from one system to the other. If you can have both systems open in separate browser windows side-by-side, that will make it easier to perform the steps below successfully.

Beacon part 1

For this part, you will need a configured Beacon CMS (customer management system) instance.

  1. In the Beacon CMS, configure Evergent as a Store.
  2. On the Settings > Location page for the store, define the countries that the service will be available in.
  3. On the Settings > Currencies page for the store, define the currencies for each country.

Evergent part 1

For this part, you need to set up BUs (business units) in the Evergent Go Portal. Each Evergent BU will map to a specific country or region of the world, and these, and the currencies associated with them, should match what you entered into the Beacon CMS.

  1. Configure the currency of the BU.
  2. Configure the country where the BU is located
  3. Configure the payment gateways and payment methods for the BU:
    • Credit Card, Debit card
    • E-wallets
    • In-app purchases (iOS, Google Play, Amazon Billing, Roku Pay)

Beacon part 2

Evergent BUs IDs and country mapping need to be configured per Beacon customer. This is done in the Monetization Payment Service which is a backend service owned by the Playback team. (The plan is for this config to move to Jackie Pages by the end of Q1 21). These configs should be tied to a specific Video Cloud customer account that is being used for Beacon.

Beacon part 4

An Evergent salt for that specific Beacon customer needs to be configured in the Monetization Payment Service.

Beacon part 5

The Monetization Payment Service generates a "Beacon_Monetization_Api” endpoint (beacon-monetization-service.brightcove.com) that needs to be entered into the customers Beacon CMS instance. This should be entered in the Settings page under "Payment Gateway Configuration” for Evergent.

Beacon part 6

  1. Create SVOD plans in the Beacon CMS Commerce tab. Define the monetization policies:
    1. Billing Frequency
    2. Trial period
    3. Renewal policy
    4. Select which Stores the plan will show up in:
      • Evergent
      • App stores (iOS, Google Play, Amazon, Roku)
    5. Set the devices and locations.
    6. Set the Availability dates.
    7. Configure the App Stores:
      1. Configure the availability dates.
      2. Configure the devices.
      3. Enter the price and currencies.
      4. Create the store IDs. This ID needs to be copied and pasted into the product sku in the app stores. Also, it needs to be entered in the Evergent Go for the matching SVOD plan.

Evergent part 2

In Evergent Go, create matching SVOD plans. It's important that the monetization policies match identically to what was entered in Beacon’s CMS. Also, note if you want to sell SVOD plans in different currencies, you need to use multiple Evergent BUs. The plans in each BU are mapped to Beacon using SKUs that are entered in the "Evergent Store” in the Beacon CMS.

Beacon part 7

Configure Beacon Evergent Store for your SVOD plans:

  1. Configure the availability.
  2. Configure devices (right now only web).
  3. Configure the Price and Currencies. The Currencies should map to the plans in the specific BUs that use that currency. For example, if you have a plan you are selling in EU using Euros, the Evergent ID should come from the EU Evergent BU. The Evergent ID field is where you enter the Evergent SKU.
