Create Native App

This configuration is applicable to all app platforms with the exception of Web (the next section provides steps for web apps):

  1. Click on the Applications menu entry in your tenant dashboard.
  2. Enter a name and select Native as the type and then click create.
  3. On Allowed Web Origins add file://*
  4. On Allowed Origins add file://*
  5. On Allowed Callback URLs and Allowed Logout URLs add (separated by comma):
    iOS:
    [BUNDLE_IDENTIFIER]://[AUTH0_URL]/ios/[BUNDLE_IDENTIFIER]/callback

    Example:

    com.brightcove.telemagic://sso.telemagic.tv/ios/com.brightcove.telemagic/callback
    Android:
    [ANDROID_PACKAGE_NAME]://[AUTH0_URL]/android/[ANDROID_PACKAGE_NAME]/callback

    Example:

    tv.telemagic://sso.telemagic.tv/android/tv.telemagic/callback
  6. Click on Show Advanced Settings and then Grant Types and make sure only Authorization Code, Refresh Token and Device Code are selected.
  7. On Refresh Token Behavior, Rotating can be selected (optional).

Create Web App

  1. Click on the Applications menu entry in your tenant dashboard.
  2. Click on Create Application.
  3. Enter a name and select Single Page App as the type and then click create.
  4. On Allowed Callback URLs add the following beacon web app URL: [BASE-URL]/cc.
  5. On Allowed Logout URLs add the following beacon web app URL: [BASE-URL].
  6. Click on Show Advanced Settings and then Grant Types and make sure only Authorization Code and Refresh Token are selected.
  7. On Refresh Token Behavior, Rotating needs to be selected (this will prevent some of the security Auth0 warnings). Reuse Interval in seconds can be set to 10.

Create Api service

In order to let Auth0 know that it needs to generate an access token, we need to create an Api service:

  1. Click on the APIs menu entry in your tenant dashboard.
  2. Click on Create API.
  3. Enter a name, Identifier and select RS256 as the Signing Algorithm and then click create. Identifier should be (as recommended by Auth0) a URL. This Identifier will be used as the audience when requesting access tokens by the applications.
  4. Once created, scroll down to Access Settings and make sure Allow Offline Access is enabled (required to be able to generate the Refresh Token).
