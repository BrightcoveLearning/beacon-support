Introduction

To provide integration between customer user management systems (UMS) and Beacon, Brightcove has implemented Open ID Connect (OIDC) for the most reliable way of providing authentication.

The Open ID Connect (OIDC) is a layer on top of the OAuth 2.0 protocol for external authentication for services like Auth0 and It allows clients to verify the identity of the end user based on the authentication performed by an Authorization Server.

Here are the prominent features of our OIDC integration:

  • Existing 3rd party databases can be used to authenticate Beacon users.
    •   
  • Beacon Users register, authenticate and change passwords through their Beacon apps.
  • Beacon apps support in app purchases.
    •  
  • Existing applications and Beacon apps share the same authentication database.
  • New viewers can register in the 3rd party system.
  • Device pairing workflows (2nd device, enter code on screen to auth the TV app).
  • Social logins if they are supported by the OIDC provider.
  • Beacon Stream concurrency limiting is supported.
  • The OIDC provider may offer additional Authentication methods beyond User name & psw (multifactor, passwordless, device pairing, google account pairing).

Implementing an OIDC Auth0 Provider

Here are the detailed steps to follow in Brightcove Beacon to implement the OIDC Auth0 Provider:

  1. Enter to your Brightcove Beacon account.
  2. Click on the gear button at the top right.
  3. Click the Auth Providers button on the left menu.
  4. Add the Auth Endpoint: /account/social_login .
  5. For the Auth Type field, select openid_connect.
  6. Fill the fields for each of the OpenID Connect settings:
    Most of the information to fill in the fields for this form is in your Auth0 account.
    • A) Base Url: Please review the value for this field with your Brightcove representative.
    • B) Revoke Refresh Url: Set as oauth/revoke
    • C) Pairing Codes Url: Set as oauth/device/code
    • D) Has Audience: Please review the value for this field with your Brightcove representative.
    • E) User Info Url: Set as userinfo
    • F) Access Token Url: Set as oauth/token
    • G) Logout Url: Set as v2/logout
    • H) Jwks Location: Is the JSON Web Key Set located in the Advanced Settings > Endpoints tab from your application in your Auth0 account.
    • I) Refresh Token Url: Set as oauth/token
    • J) Authorization Url: Set as authorize
    • K) Scopes: Please review this field with your Brightcove representative.
  7. Click the Save Settings button.
