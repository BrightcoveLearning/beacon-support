Overview

To stream a Live event to your Application, you must follow a series of steps to connect your Video Cloud Live account and Brightcove Beacon.

A high-level view of the process is:

  1. In Brightcove Beacon, create an event and a playlist.
  2. In Video Cloud, access the Live module to create your event and obtain a job ID.
  3. In Brightcove Beacon, register your live event using the job ID from Video Cloud.
  4. The Live video will start streaming in the Brightcove Beacon App.

In the last section of this document you will see the steps to create a different type of event in Live by using a Static Entry Point. This allows for recurring live events that can be activated and deactivated and need to be configured only once.

The rest of this document will provide details for each of the steps above.

Creating a Live event in Brightcove Beacon

Follow these steps to create an Event in Brightcove Beacon.

Only activated accounts with Brightcove Beacon Live will have the Playlists and Events available in the tabs section. Please contact Brightcove Support in case you don't have them activated.
  1. In Brightcove Beacon, click on the Events tab to start the set-up of an Event.
  2. Click on Add a new Event.
  3. Give the Live event a Name, and an Event Start Date.
  4. In the Textual Data tab, fill your event information.
  5. Click on the Non-Textual Data tab.
  6. Supply an image for the Portrait Poster and Landscape Poster matching the requirements below. You DO NOT need to supply images for Thumbnail and Wide banner.
    Image Requirements
    Portrait Poster Poster image (2:3)
    Minimum Width: 1000px Height: 563px
    Maximum Width: 4000px Height: 2250px
    Landscape Poster Landscape image (16:9):
    Minimum Width: 1000px Height: 563px
    Maximum Width: 4000px Height: 2250px
    Note that you can have different images for English, French and German.
  7. Click the Rights & Scheduling tab, as shown here:
  8. Add values for the following fields:
    • Rights Type: Choose the type of rights between Advertising, Subscription or Free.
    • Start Time: Set the start date of your event.
    • End Time: Set the end time of your event.
    • Max Streams: Set the maximum number of streams allowed to view the live event. The default number is 0 (zero).
    • Devices to publish: Add the devices to which you wish to publish. Select the Select / Unselect All Devices to see a list of available devices.
    • Permitted locations: Select the locations where you want to stream your content, if no restrictions enter World.
    • Denied Locations: Select the locations where you DO NOT want to stream your content.
  9. Click the Create New Event button to create your event.

Adding the Brightcove Beacon event to a playlist

Once you create an event, the next task is to add the event to a Playlist. The following steps lead you through that process:

  1. In Brightcove Beacon, click the Playlists tab.
  2. Click on the Add new playlist button.
    You need to create a playlist in Brightcove Beacon that will contain your Live events. Do not use a playlist from Video Cloud, using a Video Cloud playlist may overwrite and delete your Brightcove Beacon Live recordings.
  3. In Playlists, from the content type tabs, select the Events tab.
  4. Look for your event in the contents list and click on the (+) button to add it to the left playlist. You can position the order of the events in the way you want them to be displayed in the app.
  5. Adjust the settings of your playlist (time, devices, location, etc.) in the Rights & Scheduling tab, as you did with your event.
  6. Click on the Save changes in Playlist button.
  7. Click on the Tools (wrench) button in the upper right of the screen. Then clear the cache.

At this point, your event is created as a container in the Brightcove Beacon app as shown in the left image (the event names and pictures on the image are for example purposes only), click on your event, and you will see something similar to the image on the right. To actually start streaming a Live video, you will need to set up your transmission in the Video Cloud Live Platform.

Creating a Live event in Studio

Follow these steps to create the actual live event that will be streamed to Brightcove Beacon:

  1. Log in to Video Cloud Studio and click Live the navigation header.
  2. From the Manage Events page, click Create Event. The Create Live Event or Create Live Channel page will open. The properties when creating events and channels are the same. The screenshot in this document covers creating an event.
  3. Enter the event details:
    • Event Name - Name for the event; this name will also appear in the Media module if the video is saved as a video asset (max 255 characters)
    • Short Description - Description of the event (max 250 characters)
    • Tags - Tags to associate with the event; tags will be assigned to the video if the video is saved as a video asset (max 128 characters and 1200 tags)
    • Reference ID - Reference ID for the event; must be unique (max 150 characters)
  4. Any required custom fields for the account will be displayed. Assign values to the custom fields.
  5. Click Allowed if you wish to enable the live event to be streamed to social media platforms, otherwise click Not allowed. This can be done once the event has started streaming or by using the Social module.
  6. Select a Live Ingest Profile. The dropdown list will list all Live ingest profiles. The ingest profile will control the quality and number of live renditions that are created. Clicking the Manage Profiles link will open the ingest profile editor in another browser tab. Note that you must be an account administrator to access the ingest profile editor. Click the RENDITIONS link to view the renditions that will be created using the selected ingest profile.
    The bitrate and billing details of the three tiers are as follows:
    • Tier 1 allows up to 5,500 Kbps and is charged at 1.0x hours
    • Tier 2 allows up to 10,000 Kbps and is charged at 1.5x hours
    • Tier 3 allows up to 20,000 Kbps and is charged at 2.0x hours
  7. Select Convert event to video asset when complete to save a VOD version of the live video to your Video Cloud account. When this option is selected, the highest quality rendition from the selected Live Ingest Profile is used as the source for the VOD video. The default ingest profile for the account is used when creating renditions. Note that the VOD version is not available as soon as the live event ends, the video will be available once the ingestion process is complete.
    Note: This option is not supported when using a recurring event.
  8. Click ADVANCED OPTIONS and configure the advanced options.
    Note: For Brightcove Beacon Live work, you have to enable Stream Status Notifications checkbox in the Advanced options.
  9. Click in the Enable stream status notifications option and add the notifications URL from Brightcove Beacon CMS (your Brightcove Support representative will provide this URL).
  10. Click Create Event to start the live event. The Control Room page will open with details about the live event including the Streaming Endpoint (RTMP URL) and Stream Name which will be used by your encoder. It will take about a minute to process your streams.
    To start streaming in the Live platform, you will need an Encoder, for information on how to use an encoder please read the following the documentation:

    Step-by-Step: Using the Live Module with Telestream Wirecast

    Step-by-Step: Using the Live Module with Open Broadcaster Software (OBS)

  11. In the Control Room copy the Job Id and the Video ID from the Test Event panel, you will need both to configure the Live event in the Brightcove Beacon platform settings.
  12. Next, go back to Brightcove Beacon and click in the Event tab, select the event you previously created there, and in the Basic Data settings paste the Job ID of your Live event.

  13. Go to the Streams tab and create a new stream by clicking in the + Add new stream button.

    In the Stream Video ID field, paste the Video ID from the Video Cloud Event you created before.

    Fill in the following information:

    • Select the number of devices in which you can publish your video.
    • Change the Stream Type to: Live
    • And click the Update event button before leaving the page:
  14. Click on the Tools (wrench) button in the upper right of the screen. Then clear the cache.
  15. Open your Brightcove Beacon App and select your event (the event names and pictures on the image are for example purposes only).
  16. Click on the Play button, and you should be able now to see your Live streaming from Video Cloud in the Brightcove Beacon App.

Once your event ends, you can save it as a clip to transmit it later as a VOD (Video On Demand) on Brightcove Beacon.

Creating a Clip from your Live event

You have to create a clip of your Brightcove Beacon Live event if you want to keep the video for streaming later on as a VOD recorded video.

The Live module provides the ability to create video clips from completed events. Clips can also be created while a live event is in progress.

For a complete Clipping feature list, please review the Creating a Video Clip Using the Live Module documentation.

  1. In the Live module go to the Create Clip tab in the left panel
  2. Using the clipping buttons or shortcut keys, set the start and endpoints for the clip. Or just click on the Save button to keep the entire video.
  3. Enter the clip video properties.
    • Clip Name - Name of the clip
    • Short Description - Description of the clip
    • Tags - Tags to associate with the clip
    • Reference ID - Reference ID to assign to the clip
    • Remove Source Ad Break Content - This option removes ad breaks triggered within the clip, keeping only program content. Note: The Remove Source Ad Break checkbox will only appear when the stream is SSAI enabled.
  4. Click Create Clip. The transcoding process for the clip will begin. Clips are transcoded using the default ingest profile for the account.

Adding your Clip to Brightcove Beacon as a playable stream video.

  1. Take the number ID of your created clip:
  2. In the Brightcove Beacon platform, go to the Events tab and select the Streams menu, change the Stream Type in the dropdown from Live to Stream, and in the Stream Video ID option, paste the ID number you just copied from your Clip event in the Live platform.
  3. Click in the Basic Data tab and change the Status from Ended to Finished. Before leaving click the Update Event button at the bottom of the page, this will make the Play button available in the App to see the recording of the event.

(Static Entry Point) SEP Live events

SEP are recurring events that you can let activated/deactivated when your event starts over and over again as for a weekly program, for example. This event can be used as a one for all Live Job ID that can be transmitted to several channels in the Brightcove Beacon App.

  • Note that when you create a recurring event, a video is automatically created for it in the Media module. Do not delete this video, as it will be needed every time the event recurs.
  • The recurring event should be activated before the encoder is started.

To create a recurring event, select the Create this as a recurring event using a Static Entry Point (SEP) option when creating a live event and then click Create Event.

  1. In the Live Platform Create a new Event. For more information in how to create an event please read the following documentation.
  2. Expand the Advanced Options, and enable the Create this as a recurring event using a Static Entry Point (SEP) option. For more information in SEP click here.
  3. When you already have a SEP event, click on the Standby tab, and you will be able to activate/deactivate it from the Live Control Room.
  4. Copy your Live Job ID
  5. Return to the Brightcove Beacon platform and configure your events giving them a name and a scheduled time.
  6. The date and time of your event must be in the following format:
  7. Click in the Update event button.

You can perform this process with all the event channels you want in the Brightcove Beacon platform, and all of them will be streaming from the same Live event transmission, but each one with the different date and time that you adjusted in each of its event settings.
