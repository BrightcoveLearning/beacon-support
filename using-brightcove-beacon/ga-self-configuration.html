Introduction

The configuration process consists of the following tasks:

  • Create four Google Analytics properties
  • Create a set of custom dimensions and one custom metric for each property.
  • Create four Google Tag Manager containers.
  • In Google Tag Manger configure each container.

Required tools

Brightcove uses various tools to implement the collection, manipulation and display of analytics data using Google Analytics. These tools are:

  • Google Analytics - A web analytics platform offered by Google as part of their Google Marketing Platform brand.
  • Google Tag Manager - First, understand that tags are snippets of code which are added to a site to collect information and send it to third parties. Tags are use for numerous reasons, including scroll tracking, monitoring form submissions, conducting surveys, etc. With Google Tag Manager you can, according to Google, "manage and update tags quickly and make more informed decisions faster. Tag Manager integrates with all Google and third-party tags, and gives you complete control over when and how your tags fire."
  • Firebase - Firebase is a platform developed by Google for creating mobile and web applications. Firebase includes the Firebase Realtime Database, which is a cloud-hosted database.
  • Google Data Studio - From Google: "Data Studio is a free tool that turns your data into informative, easy to read, easy to share, and fully customizable dashboards and reports."

Create Google Analytics properties

You need to create the following four properties:

  • Beacon Android
  • Beacon iOS
  • Beacon STV
  • Beacon Web

The following steps guide you through creating the Beacon Android property. You will need to create the others following the same process.

  1. Log in to your Google Analytics account.
  2. Click the Admin (gear) icon on the bottom-left of the screen.
  3. On the top of the middle column click Create Property.
  4. In the Property setup, supply the Property name, for the first property, use Beacon Android.
  5. Supply the Reporting time zone.
  6. Supply the Currency.
  7. Click the Show advanced options link.
  8. Turn on Create a Universal Analytics property.
  9. Enter your website's URL.
  10. Click the Create a Universal Analytics property only radio button.
  11. Check to be sure your property information is filled in similar to the following:
  12. Click Next.
  13. Complete the Business information form with your data.
  14. Click Create.
  15. Read any details you wish about your new property.
  16. Click the back arrow, highlighted in the following screenshot:
  17. Just below the Create Property you clicked earlier, click the down arrow to check to be sure your property was correctly created.
  18. Perform the proceeding steps to create the three other required properties:
    1. Beacon iOS
    2. Beacon STV
    3. Beacon WEB

Create custom metric and dimensions

Now for each property you need to create a set of custom dimensions and one custom metric.

This section will be organized in the following manner:

  • Steps will be shown how to create a single custom dimension.
  • Steps will be shown how to create a single custom metric.
  • A table will show the dimensions required for the properties Beacon Android and Beacon iOS.
  • A table will show the dimensions required for the properties Beacon STV and Beacon WEB.
  • A table will show the single metric that all four properties must have.

You will then need to create the dimensions and metric for each of the four properties.

Creating a custom dimension

  1. Be sure you are still logged in to Google Analytics.
  2. Click on the dropdown (highlighted in yellow in the screenshot) to see all your Google Analytics account and property.
  3. From the three-paneled dropdown, select:
    • Your Brightcove Beacon account
    • The property for which you will be creating dimensions/metric
    • Your view (you will only have one view per app)
  4. Click the Admin (gear) icon on the bottom-left of the screen.
  5. Near the bottom of the middle column, under Custom Definitions click Custom Dimensions
  6. Click NEW CUSTOM DIMENSION.
  7. For the Name enter Episode Number.
  8. From the Scope dropdown select Hit.
  9. Leave the dimension Active.
  10. Click Create.
  11. From the example code screen click Done.
  12. You will see your newly created dimension in a list, as shown here:
    It is crucial that the Index matches the dimension name as shown in the tables below. To do this, simply create the dimensions in the order shown in the tables.
  13. To create another dimension click NEW CUSTOM DIMENSION.

Creating a custom metric

  1. Be sure you are still logged in to Google Analytics.
  2. Click on the dropdown (highlighted in yellow in the screenshot) to see all your Google Analytics account and property.
  3. From the three-paneled dropdown, select:
    • Your Brightcove Beacon account
    • The property for which you will be creating dimensions/metric
    • Your view (you will only have one view per app)
  4. Click the Admin (gear) icon on the bottom-left of the screen.
  5. Near the bottom of the middle column, under Custom Definitions click Custom Metrics
  6. Click NEW CUSTOM METRIC.
  7. For the Name enter Time Watched.
  8. From the Scope dropdown select Hit.
  9. For the Formatting Type select Time.
  10. Leave the dimension Active.
  11. Click Create.
  12. From the example code screen click Done.
  13. You will see your newly created metric in a list.
This is the only custom metric you will need to create for each of the four properties.

Configuring Google Tag Manager

In this section of the document you will perform the following tasks:

  • Create four containers, one for each property created earlier.
  • Import the configuration JSON files which were emailed to you are part of the Obtain the templates and configuration files section above.
  • Enter your tracking ID, copied from your Google Analytics account, into each of the containers variables.
  • Publish your containers.

Create containers

  1. Review Google's documentation to Setup and install Tag Manager.

  2. In Tag Manager, create an account or use an existing account. To create a new account, select the Create Account button.

    By default, a new container is created for a new account. You will create additional containers within each account.

  3. Enter values for your new account page:

    • Account Name: Your company name, or perhaps Beacon.
    • Country: Home country for your company.
    • Container Name: Beacon WEB You will be creating a container for each property.
    • Target Platform: Web
  4. Click Create.
  5. Click OK to close the dialog that opens.
  6. From the Workspace, click Admin.
  7. Click the plus sign to add another container.
  8. Complete the form and click Create to create the iOS container.
  9. Be sure the following four containers are created with the correct Target Platform.
    Container Name Target Platform
    Beacon Android Android
    Beacon iOS iOS
    Beacon STV Web
    Beacon WEB Web
  10. From the Workspace, use the account dropdown at the top of the page to be sure all containers have been created.

Import configuration files

Perform the following steps to import the configuration files for your containers:

  1. From the Workspace, click Admin.
  2. Check which container you are now configuring, then click Import Container.
  3. Click Chose container file, and select the correct JSON file for this container.
  4. For Choose workspace, click Existing. Then from the dialog that opens select Default Workspace.
  5. For Choose an import option, click the Overwrite radio button.
  6. You will see information about tags, triggers, variables and templates.
  7. Click Confirm, and you will be taken back to the Workspace..
  8. In the Workspace, click on Tags, Triggers and Variables to confirm the import was successful.
  9. You now need to import the correct JSON file for each container.

Enter tracking ID

You will now copy from Google Analytics the tracking ID for each property, then paste it into the corresponding container.

  1. Log in to Google Analytics.
  2. Use the account/property selector at the top of the page to choose the Beacon Android property.
  3. Click Admin.
  4. From the Property column first click Tracking Info the Tracking Code.
  5. Copy the Tracking ID.
  6. Return to Google Tag Manager.
  7. Be sure you are editing the Beacon Android container.
  8. Click Variables from the left side navigation.
  9. Click the Tracking Code variable.
  10. Paste the copied tracking ID from Google Analytics into the Tracking ID form field.
  • You now need to paste the tracking ID into the Tracking Code variable in each container.

    • Publish each container

    The last step you need to do in Google Tag Manger is submit each container for publishing. The following steps guide you through that process:

    1. Return to your Workspace.
    2. Note the container you are in.
    3. On the top right of the UI, click Submit.
    4. Be sure the Publish and Create Version button is selected.
    5. If you choose, add a version name and version description.
    6. Click Publish.
    7. If you did not provide a version name and description you be asked again. Skip if you are sure you do not want those.
    8. Check to be sure the information about your container is correct.
  • You now need to be sure all containers are published.
